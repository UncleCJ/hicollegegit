Initializing a project directory

It is first command

git init

you will use to initialize a git repository. Most of other git commands are not available outside of an initialized repository.

Running this command it will create a .git subdirectory in the project root that contains all the other necessary metadata for the repo

Cloning a repository

git clone /path_to_repository

Git configuration

To get the list of your git configuration

git config --list

Identify yourself in GIT by setting username or email

git config --global user.name git config --global user.email

Git history

The most powerful and basic command to figure out where bugs were introduced amd revert any problematic changes.

git log

See the contribution of every author and number of commits they made

git shortlog -n

Show history for only an amount of commits

git log -#numberofcommits (i.e. git log -1)

Decorate the commits history by commit id and message

git log --oneline

Show only commits from a branch that are not merged into a branch

git log --no-merges

See the history changes on a file

git log -p "filename"

Decorate the history of commits by also showing the branches and tags

git log --oneline --decorate"

Decorate the history of commits by also showing the branches and tags

git log -S "text to search"

Show the history of commits from specific such as after or before (date , time or day)

git log --after="date, time ,day"

See the author of commit that is responsible of every line of code on a file

git blame "fileName"

Git commits

Adding files to the staging area before commit

git add "-fileName-"

Removing untracked files before commit

git clean -f"

Move a commit from a branch to another branch

git cherry-pick #commitID"

Moving untracked files from a branch to another branch before commit

git stash save "stash description" git stash apply

Committing and staging untracked file at the same time

git commit -a -m "message to commit"

Editing the commit message

git commit --amend -m "your message"

Git Branching

git checkout branch

To show local branches

git branch"

Delete a branch without unmerged changes otherwise to force

git branch -d" git branch -D

Rename a branch

git branch -m "new branch name"

Show all remote branches

git branch -r

Delete a remote branch

git branch -d -r origin/branch_name

List the branch that are merged or check if a branch is merged to another branch. Same applies for remote branches (by adding -r and origin/branch_name)

git branch --merged git branch --merged branch_name

Git merge & pull

With git pull, GIT automatically merges the commits without letting you review them so if you do not have control over your branches then you may run in frequent conflicts

git pull

While with git fetch and merge, you have more control over your commits that brings them update to date from remote to local repo and then to integrate merge them to local repo

git fetch git merge
